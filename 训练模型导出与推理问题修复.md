## 1. 场景背景
*   **模型**: Qwen2.5-VL-7B-Instruct
*   **硬件**: 2x Nvidia A800 (80GB)
*   **环境**: AutoDL (System RAM 50GB)
*   **训练方式**: DeepSpeed ZeRO-3 + LoRA (SFT)
*   **目标**: 将训练生成的 DeepSpeed Checkpoint 转换为可直接加载的 HuggingFace 格式模型进行推理。

## 2. 遇到的问题

### 问题 A：推理加载时 GPU 飙升后卡死 (Hang)
*   **现象**: 使用 `zero_to_fp32.py` 转出的 `.bin` 文件推理时，GPU 占用率瞬间飙升至 100% 后立即掉回 0%，程序彻底卡死，无报错。
*   **原因**:
    1.  **文件格式**: `.bin` (Pickle) 格式加载需要大量 CPU 内存进行反序列化。
    2.  **精度过大**: DeepSpeed 默认导出 FP32 权重 (约 30GB)，系统内存 (50GB) 在处理加载和多卡切分时发生溢出或死锁。
    3.  **多卡死锁**: `accelerate` 的 `device_map="auto"` 在双卡 A800 环境下处理非 Safetensors 格式大模型时容易发生通信死锁。

### 问题 B：加载权重时报 Key Mismatch 错误
*   **现象**: 报错 `Missing key(s)` 和 `Unexpected key(s)` (包含大量 `base_model.model...` 和 `lora_A/B`)。
*   **原因**:
    1.  **结构不匹配**: DeepSpeed 保存的是**带有 LoRA 包装层**的模型结构。
    2.  **直接加载错误**: 尝试将 LoRA 结构的权重直接硬加载到干净的 `Qwen2_5_VL` 基础模型中，导致层级名称对不上。

---

## 3. 最终解决方案 (Golden Path)

我们需要编写一个专用的 Python 脚本，完成以下**四个核心步骤**：
1.  **内存优化读取**: 读取 ZeRO-3 的 FP32 权重，**只提取 LoRA 部分**，随即释放巨大内存。
2.  **Key 名称清洗**: 修正 DeepSpeed 引入的 `base_model.model.model` 多余前缀。
3.  **Peft 合并**: 加载 BF16 基础模型，挂载提取出的 LoRA 权重，执行 `merge_and_unload()`。
4.  **格式转换**: 将最终模型保存为 **Bfloat16** + **Safetensors** 格式。

### 步骤一：创建转换脚本 `convert_lora_final.py`

```python
import os
import torch
import gc
from transformers import AutoConfig, Qwen2_5_VLForConditionalGeneration, AutoProcessor
from peft import PeftModel, LoraConfig
from deepspeed.utils.zero_to_fp32 import get_fp32_state_dict_from_zero_checkpoint

# ================= 配置 =================
checkpoint_dir = "/root/autodl-tmp/checkpoint"  # DeepSpeed checkpoint 目录
base_model_path = "/root/autodl-tmp/Qwen2.5-VL-7B-Instruct" # 原模型目录
output_dir = "/root/autodl-tmp/Qwen2.5-VL-7B-Final" # 输出目录

# LoRA 配置 (需与训练代码一致)
LORA_RANK = 8
LORA_ALPHA = 32
LORA_DROPOUT = 0.05
TARGET_MODULES = ["q_proj", "k_proj", "v_proj", "o_proj", "gate_proj", "up_proj", "down_proj"]
# =======================================

def clean_memory():
    gc.collect()
    torch.cuda.empty_cache()

try:
    print("1️⃣ 正在读取 DeepSpeed Checkpoint (FP32)...")
    state_dict = get_fp32_state_dict_from_zero_checkpoint(checkpoint_dir)

    print("2️⃣ 提取并清洗 LoRA 权重...")
    lora_state_dict = {}
    for key, value in state_dict.items():
        # 仅提取 LoRA 相关参数
        if "lora_" in key or "modules_to_save" in key:
            # 修正 DeepSpeed 引入的多余前缀
            new_key = key.replace("base_model.model.model.", "base_model.model.")
            lora_state_dict[new_key] = value
    
    # 关键：立即释放 30GB+ 的原始权重，防止内存溢出
    del state_dict
    clean_memory()

    print("3️⃣ 加载基础模型 (BF16)...")
    model = Qwen2_5_VLForConditionalGeneration.from_pretrained(
        base_model_path,
        torch_dtype=torch.bfloat16,
        device_map="cpu", # 在 CPU 进行合并
        trust_remote_code=True
    )

    print("4️⃣ 应用 LoRA 并加载权重...")
    peft_config = LoraConfig(
        task_type="CAUSAL_LM", inference_mode=True,
        r=LORA_RANK, lora_alpha=LORA_ALPHA, lora_dropout=LORA_DROPOUT,
        target_modules=TARGET_MODULES
    )
    model = PeftModel(model, peft_config)
    model.load_state_dict(lora_state_dict, strict=False)

    print("5️⃣ 合并模型并保存 (Safetensors + BF16)...")
    model = model.merge_and_unload()
    model = model.to(dtype=torch.bfloat16)
    
    model.save_pretrained(output_dir, safe_serialization=True)
    
    # 别忘了保存 Processor
    processor = AutoProcessor.from_pretrained(base_model_path, trust_remote_code=True)
    processor.save_pretrained(output_dir)

    print(f"✅ 完成！模型已保存至: {output_dir}")

except Exception as e:
    print(f"❌ 错误: {e}")
```

### 步骤二：运行转换

```bash
python convert_lora_final.py
```

### 步骤三：推理脚本 `inference_final.py`

注意：为了避免 `accelerate` 库在双卡 A800 上对小模型切分产生 Overhead 或死锁，**强烈建议显式指定单卡运行**。

```python
import torch
from transformers import Qwen2_5_VLForConditionalGeneration, AutoProcessor
from qwen_vl_utils import process_vision_info

# 指向转换后的目录
model_dir = "/root/autodl-tmp/Qwen2.5-VL-7B-Final"
device = "cuda:0" # 强制单卡

print(f"🚀 加载模型: {model_dir}")
model = Qwen2_5_VLForConditionalGeneration.from_pretrained(
    model_dir,
    torch_dtype=torch.bfloat16,
    device_map=device, 
    trust_remote_code=True
)
processor = AutoProcessor.from_pretrained(model_dir, trust_remote_code=True)

# 测试
messages = [{"role": "user", "content": [{"type": "text", "text": "你好"}]}]
text = processor.apply_chat_template(messages, tokenize=False, add_generation_prompt=True)
image_inputs, video_inputs = process_vision_info(messages)
inputs = processor(text=[text], images=image_inputs, videos=video_inputs, padding=True, return_tensors="pt")
inputs = inputs.to(device)

print("生成中...")
generated_ids = model.generate(**inputs, max_new_tokens=128)
output_text = processor.batch_decode(generated_ids, skip_special_tokens=True)[0]
print(output_text)
```

## 4. 关键经验总结

1.  **DeepSpeed 与 LoRA**: DeepSpeed 保存的 checkpoint 包含了 LoRA 的 wrapper 结构。如果想转为 HF 格式，必须使用 `PeftModel` 加载并 `merge`，不能直接 `load_state_dict`。
2.  **Safetensors 优于 Bin**: 永远优先使用 `safe_serialization=True`。`.bin` 格式在加载大模型时极易导致内存爆炸或死锁。
3.  **BF16 优于 FP32**: 在 A800 等安培架构显卡上，始终将模型转为 `bfloat16` 保存。这能节省一半显存和磁盘空间，且加载速度快一倍。
4.  **Device Map**: 对于 7B 这种中小模型，在 80G 显卡上，**不要**使用 `device_map="auto"`，直接指定 `cuda:0` 最稳定。
